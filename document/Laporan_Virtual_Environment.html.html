Laporan Pembuatan Virtual Environment (venv) di Server

1. Informasi Server

  • Hostname: agung@192.168.1.8
  • OS: Ubuntu 24.04.1 LTS
  • Kernel: Linux 6.8.0-41-generic x86_64
  • System Load: 0.04
  • Usage of /: 37.7% of 11.21GB
  • Memory Usage: 8%
  • IPv4 Address: 192.168.1.8

2. Tujuan

Membuat lingkungan virtual (virtual environment/venv) untuk isolasi pengembangan aplikasi berbasis Python di server.

3. Langkah-Langkah

1. Persiapan Awal

Masuk ke server melalui SSH.

python3 --version

Jika Python 3 belum tersedia, instal menggunakan:

sudo apt update
sudo apt install python3

2. Instalasi Paket python3-venv

Pastikan modul venv sudah diinstal untuk membuat virtual environment:

sudo apt install python3-venv

3. Pembuatan Virtual Environment

Buat virtual environment baru:

python3 -m venv venv

4. Mengaktifkan Virtual Environment

Aktifkan virtual environment:

source venv/bin/activate

Setelah aktif, prompt akan menunjukkan nama virtual environment, contoh:

(venv) agung@agung-hendi-temorubun:~$

5. Keluar dari Virtual Environment

Nonaktifkan virtual environment:

deactivate

6. Penghapusan Virtual Environment

Hapus folder virtual environment:

rm -rf venv

4. Catatan Tambahan

Gunakan virtual environment untuk memastikan paket dan konfigurasi tetap terisolasi dari sistem global.

Untuk memastikan keamanan dan performa, selalu gunakan virtual environment dalam proyek berbasis Python.

5. Hasil

Virtual environment berhasil dibuat, dan server siap digunakan untuk pengembangan aplikasi Python dengan isolasi lingkungan.

Dokumentasi selesai. Jika ada kesalahan atau langkah tambahan, silakan hubungi administrator.
